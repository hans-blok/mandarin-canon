# Artifact Type Matrix Per Value Stream Phase

---

## Provenance Accountability

This normative artifact is derived on the basis of the following consulted sources:

**Consulted sources**:
- `constitutie.md` (version 2.3.0, as the normative framework for the value stream concept)
- `mandarin-domeinconcepten.md` (version 2.13.0, for the definition of artifact type)
- `togaf.kaderdefinitie.md` (version 2.0.0, as the frame source for AOD phasing)
- `temp/artefact-type-overzicht.md` (generated 2026-04-15, as the source for artifact type ID assignment)

**Explanation**:
This document records which artifact types are recognized within the Mandarin ecosystem, including their unique artifact type ID. Per value stream phase, it presents a matrix that shows which artifact types can be produced in that phase.

---

## 1. Introduction

### 1.1 - Purpose of This Document

This document describes the canonical set of **artifact types** that are recognized within the Mandarin ecosystem, together with their unique **artifact type ID**. Per **value stream phase**, it presents a matrix that makes the relationship between phases and artifact types explicit.

### 1.2 - Structure

- **Section 2**: Overview of all artifact types with their ID and explanation
- **Sections 3-8**: Matrices per value stream phase

## 10. Relationship to Other Documents

This document is normative and serves as a source for:
- **Agent contracts**: When an agent contract is prepared, the artifact type ID is included as an FK
- **Templates**: The template ID refers to the artifact type ID that the template supports
- **Artifact headers**: The artifact type ID is a required field in the YAML header

This document refers to:
- **mandarin-domeinconcepten.md** (v2.13.0) for the definition of "artifact type"
- **constitutie.md** (v2.3.0) as the highest normative source
