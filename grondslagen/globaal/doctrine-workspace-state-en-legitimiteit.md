## 1. Rol van de Workspace State

Elke workspace beschikt over één en slechts één **Workspace State**  
(`state-<workspace-naam>.md`).

Deze state is het **enige geldige startpunt** voor betekenisvolle actie
binnen de workspace.

De state beschrijft niet slechts context,
maar **legitimeert handelen**.

---

## 2. Verplichte leesregel

Geen enkele agent mag:

- een artefact creëren
- een artefact wijzigen
- een afgeleid resultaat produceren
- een norm toepassen

zonder **voorafgaand** de geldende workspace state te hebben gelezen.

Deze leeshandeling is geen optimalisatie,
maar een **voorwaarde voor legitimiteit**.

Handelen zonder vooraf gelezen state is ongeldig,
ongeacht de inhoudelijke kwaliteit van de output.

---

## 3. State als actualiteitsanker

De workspace state fungeert als expliciet **actualiteitsanker**.

Zij maakt zichtbaar:
- welke canon geldt
- welke beleidsdocumenten van toepassing zijn
- welke artefacten recent zijn gewijzigd
- waar verhoogde aandacht vereist is

Agents mogen geen aannames doen over actualiteit
buiten wat expliciet in de state is vastgelegd.

---

## 4. Ondersteuning van agents (leeskompas)

De workspace state biedt actieve ondersteuning aan agents door expliciet te vermelden:

- welke artefacten recent zijn aangepast
- welke normatieve documenten opnieuw gelezen moeten worden
- welke wijzigingen impact hebben op afgeleide artefacten

Dit ontslaat agents niet van hun leesverplichting,
maar **stuurt hun aandacht gericht**.

---

## 5. Verplichte logging van wijzigingen

Wanneer een agent:

- een artefact wijzigt
- een normatief document aanpast
- een nieuw canoniek of afgeleid bestand toevoegt

 dan is die agent verplicht om:

- deze wijziging **onverwijld vast te leggen** in de workspace state
- met een feitelijke beschrijving van:
  - wat is gewijzigd
  - wanneer
  - door wie (mens of agent)

Wijzigingen die niet in de state zijn gelogd,
worden geacht **niet te hebben plaatsgevonden**.

---

## 6. Relatie tot herkomstverantwoording

De workspace state en de herkomstverantwoording vervullen verschillende, aanvullende rollen:

- De **state** bepaalt *wat gelezen moet worden*
- De **herkomstverantwoording** toont *wat daadwerkelijk is gelezen*

Output zonder expliciete herkomstverantwoording
of zonder vooraf geraadpleegde state
is canoniek ongeldig.

---

## 7. Slotbepaling

De workspace state is geen administratief hulpmiddel,
maar een **constitutioneel mechanisme**.

Zij maakt handelen uitlegbaar,
actualiteit toetsbaar
en samenwerking betrouwbaar.

Wie de state niet leest,
handelt zonder legitimiteit.
